Why Icicle?

Getting Started

Features

* Some features are only available by installing the separate modules.
** Some features are still in development, therefore may not be available for the public, or they may be buggy.

Dependency Injection

Are you used to using dependency injection in Spring, or just like it? Well, we've made our own from scratch, so you don't have to give that up.

Configurations

Easy-to-use configuration system out of the box.
You don't have to deal with making a multi-config system with reloading etc.

Translation System

Multi lingual projects made easy, with a highly customizable localization system. We even provide geolocation and a special code parser for even more solutions.

Kotlin Support

Icicle not only supports Kotlin in its core, but also in many of the separate modules we provide. We even have a dedicated one just for the juicy Kotlin-only gadgets!

Innovative Commands

Commands are the main way you interact with a plugin, so it has to be done right.
Icicle provides a very intuitive, easy-to-use, annotation-based system.

Protocol

Make an impact on people, with a unique experience achieved with packets. Choose our pre-defined presets, or build your own with our custom protocol library!

Database Interface

Soon...

Provide a wide variety of database-types for the server owners to choose from. Interact with SQL, NoSQL and file-based database solutions easily with our lightweight database handler!

Utilities

Icicle's utility classes are also available for use. They contain tools & solutions, that you don't have to copy from StackOverflow again.

Dev Tools

Assist your development with Icicle, and use our custom developer tools for
Gradle and IntelliJ.

Module-based

Icicle only downloads/updates/loads modules (dependencies), that are truly required for the projects, therefore saving precious space and making it lightweight.

Highly Customizable

Built with a customizability in mind, every part of Icicle can be customized (yes even the core). and built-in features can be replaced with your imagination.

Much More

The list just goes on. The possibilities are truly endless, especially with a whole community behind it.
Join and be part of this community!

Creating Configurations

Don't waste time coding a multi-file configuration system for the 100th time.
Save yourself some headache with our intuitive configuration system, built right into Icicle's core.

Set default values, give instructions via comments and most importantly give yourself a break from the madeness of configuration systems.

Creating Commands

Create complex interactions, or throw together a test command in a pinch with Icicle! Our command system is designed to be powerful with automatic help generation, command prediction, input validation.

Maintain your freedom with middlewares & options of customization. 


@CommandManager("demo")
public class DemoCommand {

    @Command(value = "execute", returnsTranslationKey = false)
    public String onExecuted() {
        return "You have executed the /demo execute command!";
    }

    @PlayerOnly
    @Command(value = "player", returnsTranslationKey = false)
    public String onExecuted(@CommandSender Player player) {
        return "Hm, you must be a player then! Your name is: " + player.getName();
    }

    @PlayerOnly
    @Permission("demo.admin.opCommand")
    @Command(value = "setOp", returnsTranslationKey = false)
    public String onExecuted(Player target, boolean value) {
        target.setOp(value);
        return value ? "Great, you have set " + target.getName() " to operator!"
                     :  "You have removed " + target.getName() + "'s privileges!";
    }
}
